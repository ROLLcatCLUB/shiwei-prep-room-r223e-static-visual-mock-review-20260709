# R223D Static Visual Mock Description

This is a textual static mock description, not HTML, not CSS, not a real UI.

## Default View

```text
Top: compact mode rail with selected mode.
Center: prep canvas dominates, showing current lesson plan and focused section card.
Left: source/gap rail collapsed to small indicators; high-risk confirmation remains visible if present.
Below focused section: current component suggestion strip, max 1-3 cards.
Right: binding preview closed until one component is selected.
Bottom: compact local note input with non-chat placeholder.
```

## Component Focus View

```text
Teacher selects one component card.
Right binding preview opens for selected component only.
Preview shows: media need, screen prompt, learning sheet field, teacher language example, evidence output.
Other component details remain collapsed.
```

## Quick Mode View

```text
The quick plan appears first.
Only 1-2 components appear if they protect evidence or risk.
No side rail opens by default.
Binding preview is closed until component focus.
Teacher can read the core plan in about 3 minutes.
```

## R97B Carrier Note

R223D does not modify R97B. It only says a future R97B-like carrier should prefer collapsible side rails and selected-component-only binding preview.
